# TK-SaaS：TikTok Shop 店铺运营自动化计划

## 1. 项目定位

TK-SaaS 是一个面向 TikTok Shop 店铺运营的内部自动化中控台。目标不是一开始就做“完全替代人工”的机器人，而是先把店铺每天重复、容易漏、容易超时、需要标准话术的工作集中到一个系统里，形成：

- 自动拉取订单、售后、评价、库存、达人线索等数据
- 自动识别待处理事项和风险事项
- 自动生成客服话术、售后申诉草稿、达人建联邮件草稿
- 人工确认后执行关键动作
- 用看板和提醒机制降低漏单、超时、漏回评、漏备货的问题

核心原则：**能走官方 API / ERP / 正常导出的地方，不优先做浏览器脚本硬点后台。** TikTok Shop 对 bot、scraper、自动化脚本、批量滥用交互等行为限制较严，涉及账号安全和店铺健康，必须尽量走合规路线。

### 2026-07-07 执行方向更新

项目最终目标已明确为**正常业务路径全流程无人值守执行**，而不是只生成提醒或待办：

```text
自动发现业务事件
→ 自动读取和校验数据
→ 自动执行 TikTok / ERP / 惠程荣达操作
→ 自动验证结果
→ 自动记录审计日志
→ 仅将验证码、风控、数据不确定等例外送入人工异常队列
```

接口优先级仍然是 `官方 API > ERP/API > Playwright RPA > 人工异常处理`。Playwright 是拿不到接口时的正式执行适配器，不再只是概念验证。首个纵向切片为 TikTok 待发货订单到惠程荣达订单及交接面单上传的完整闭环，详见 [全自动发货执行规格](./docs/specs/shipping-execution-automation.md)。

## 2. 当前业务流程

### 2.1 订单发货

- 订单需要 24 小时内及时揽收
- M 店下单 ST&BW 当天联系客户
- 订单送达 3 天后，邀请客户留下真实使用体验

### 2.2 售后处理

- 待退货：解释提醒客户
- 已退货：查看退货群 / 退货状态
- 不合理退款：整理证据并发起申诉
- 已有售后处理模板和客服话术

### 2.3 商品评分

- 处理差评
- 处理待回复的好评
- 差评需要客户视频或证据
- 上午查看已差评客户是否回复
- 下午处理新的评价问题

### 2.4 账号健康

- 原任务包括检查账号健康、达人健康评分
- 当前阶段暂时不做任务 4

### 2.5 达人 / 红人拓展

- 重点找“马尾”相关红人
- 联系方式主要来自红人主页公开留下的邮箱、社媒、站外联系方式
- 暂不做暴力群发，优先做筛选、建联草稿、跟进提醒

### 2.6 库存管理

- 库存平台：惠程荣达
- 上班、下班各核对一遍库存
- 库存不足时提醒备货
- 做完备货表后，对一下在途数量

## 3. 自动化优先级

| 优先级 | 模块 | 自动化价值 | 风险 | 说明 |
|---|---|---:|---:|---|
| P0 | 订单超时雷达 | 高 | 低 | 防止 24 小时未揽收，直接影响履约 |
| P0 | 售后工单池 | 高 | 中 | 自动分类、提醒、生成话术，申诉前人工确认 |
| P0 | 库存差异表 | 高 | 低 | 上下班自动核对 TikTok 库存、惠程荣达库存、在途数量 |
| P1 | 评价监控 | 中高 | 中 | 自动发现差评/待回复评价，生成回复草稿 |
| P1 | 送达 3 天评价提醒 | 中 | 中 | 只能邀请真实评价，不能诱导好评 |
| P1 | 达人线索库 | 中高 | 中 | 筛选达人、整理联系方式、生成邮件草稿 |
| P2 | 达人外联自动跟进 | 中 | 高 | 需要控制频率，避免 spam 或平台风险 |
| 暂缓 | 账号健康 / 达人健康评分 | 中 | 中 | 当前阶段不做 |

## 4. 第一阶段 MVP 范围

第一阶段不追求功能多，先解决运营每天最痛的四件事：

1. 订单是否超时
2. 售后是否漏处理
3. 库存是否需要备货
4. 评价是否需要回复

### 4.1 MVP 功能列表

#### 订单模块

- 同步 TikTok Shop 订单
- 展示订单状态：待发货、已发货、未揽收、已揽收、已送达、已完成
- 计算 24 小时揽收倒计时
- 对临近超时订单做提醒
- M 店 ST&BW 订单当天自动进入“客户联系待办”
- 订单送达 3 天后进入“真实评价邀请待办”

#### 售后模块

- 同步退款、退货、取消订单等售后数据
- 自动分为：待退货解释、已退货待核查、疑似不合理退款、已完成
- 根据售后类型匹配客服话术模板
- 自动生成申诉草稿和证据清单
- 不合理退款申诉必须人工确认后再提交

#### 评价模块

- 记录商品评价状态
- 区分差评、中评、好评、待回复评价
- 上午提醒：查看已差评客户是否回复
- 下午提醒：处理新增评价
- 差评自动生成回复草稿
- 差评需要视频时，自动生成索要视频/证据的话术草稿

#### 库存模块

- 导入或同步惠程荣达库存数据
- 导入或同步 TikTok Shop SKU 库存数据
- 对比：可售库存、锁定库存、在途数量、安全库存、建议备货量
- 上班、下班各生成一次库存核对结果
- 自动生成备货表
- 备货表生成后，提醒核对在途数量

```text

        ↓

        ↓
数据库 PostgreSQL / MySQL
        ↓
规则引擎
- 24h 揽收倒计时
- 送达 3 天评价邀请
- 售后状态分类
- 库存预警
- 差评处理提醒
- 达人跟进提醒
        ↓
AI 生成层
- 客服话术草稿
- 评价回复草稿
- 退款申诉草稿
- 达人邮件草稿
        ↓
运营中控台
- 订单看板
- 售后看板
- 评价看板
- 库存看板
- 达人看板
        ↓
通知渠道
- 飞书 / 企微 / 邮件 / Telegram / 微信提醒
```

### 6.1 前端

建议：

- React + Vite
- Tailwind CSS
- Shadcn UI / Ant Design
- 表格和看板优先，不做花哨动画

核心页面：

- 首页仪表盘
- 订单超时看板
- 售后工单看板
- 评价处理看板
- 库存核对看板
- 达人线索看板
- 话术模板管理

### 6.2 后端

建议：

- Node.js / NestJS 或 Python / FastAPI
- PostgreSQL / MySQL
- Redis：做定时任务、队列、缓存
- Cron / BullMQ / Celery：做定时同步和提醒

### 6.3 自动化平台

如果想快速验证，可以先用：

- n8n：做工作流、Webhook、通知、表格同步
- 飞书多维表格 / Google Sheets：做第一版数据中台
- Python 脚本：做库存表比对和备货表生成

## 8. 规则引擎草案

### 8.1 订单揽收规则

```text
如果订单已支付，但未揽收：
- 距离 24 小时截止还有 8 小时：普通提醒
- 距离 24 小时截止还有 4 小时：高优先级提醒
- 距离 24 小时截止还有 1 小时：紧急提醒
- 超过 24 小时仍未揽收：标记为风险订单
```

### 8.2 M 店 ST&BW 客户联系规则

```text
如果订单来自 M 店，且商品/标签包含 ST 或 BW：
- 下单当天生成客户联系待办
- 匹配对应话术模板
- 人工确认后发送
```

### 8.3 送达 3 天真实评价邀请规则

```text
如果订单已送达，并且送达时间超过 3 天：
- 生成真实评价邀请待办
- 使用合规话术：欢迎分享真实使用体验
- 不允许出现“好评返现”“请给五星”“改好评”等诱导性内容
```

### 8.4 售后处理规则

```text
如果售后状态为待退货：
- 匹配退货解释/提醒话术

如果售后状态为已退货：
- 提醒检查退货群和退货物流

如果退款原因疑似不合理：
- 自动生成证据清单
- 自动生成申诉草稿
- 人工确认后再申诉
```

### 8.5 评价处理规则

```text
每天上午：
- 检查历史差评客户是否回复
- 生成需要二次跟进的列表

每天下午：
- 检查新增差评/中评/待回复好评
- 自动生成回复草稿
- 差评需要视频时，生成索要视频证据话术
```

### 8.6 库存核对规则

```text
每天上班：
- 拉取 / 导入惠程荣达库存
- 拉取 / 导入 TikTok SKU 库存
- 生成库存差异表

每天下班：
- 再次核对库存
- 检查当天订单消耗
- 生成备货建议

做完备货表后：
- 核对在途数量
- 标记是否需要补货
```

## 9. 合规边界

### 可以做

- 订单、售后、库存、评价数据同步
- 风险提醒
- 话术草稿生成
- 人工确认后回复客户
- 达人线索整理
- 达人邮件草稿生成
- 库存表自动比对
- 备货表自动生成

### 不建议做

- 浏览器脚本自动登录 TikTok 后台乱点
- 自动批量私信达人
- 自动刷评价
- 诱导客户给五星好评
- 要求客户修改差评
- 给客户好处换好评
- 绕过平台规则爬取数据
- 高频访问达人主页导致账号风控

## 13. 当前建议

第一版不要做太大。建议先做：

1. 订单超时雷达
2. 库存核对表
3. 售后工单池
4. 评价回复看板

这四个模块做出来，就已经能明显降低漏单、漏售后、漏库存、漏差评处理的问题。达人模块可以第二阶段再上，账号健康模块当前暂缓。
